# Hero User Journey & Conversion Strategy — Sikkha (শিক্ষা বাংলা)

> **Role:** Product Manager / UX Strategist / Conversion Expert
> **Scope:** Visitor behavior analysis, journey mapping, drop-off diagnosis, conversion strategy
> **Constraint:** No UI discussion, no code, no design — only psychology and behavior
> **Assumption:** First-time visitor, 5-second decision window, mobile-primary audience

---

## Part 1: Visitor Type Analysis

### Visitor Type 1: Class 6 Student (First-Time)

**Profile:** 11-12 years old. New to the platform. Likely arrived via word-of-mouth (friend/classmate recommendation) or parent found it. Limited digital literacy. Needs permission from parent to register.

**1. Immediate goal:** "I need help with my studies — my friend said this site has free stuff."

**2. Question in first 5 seconds:** "Is this actually free? Can I use it without paying?"

**3. Information needed immediately:** Confirmation that the platform is free. Confirmation that it covers their class (Class 6). Visual proof that other kids use it.

**4. Hero should encourage:** "Browse Class 6 content" — the lowest-commitment action. Not "Register" (requires parent permission). Not "See Premium" (irrelevant to free-seeking student).

**5. What should NEVER distract this visitor:** Premium pricing, payment information, exam countdowns (Class 6 students don't have board exams yet), complex navigation, anything that requires login.

**6. Best CTA:** "ফ্রি দেখুন" (See for free) or "ক্লাস ৬ এ শুরু করুন" (Start with Class 6)

**7. Next homepage section:** Subject Explorer (to see what Class 6 content is available)

**Current drop-off risk:** HIGH. The Hero says "শিক্ষা শুরু করুন" (Start learning) and "প্রিমিয়াম দেখুন" (See premium). A Class 6 student seeing "premium" immediately worries about cost. The word "premium" is a conversion killer for price-sensitive visitors.

### Visitor Type 2: SSC Candidate (First-Time)

**Profile:** 15-16 years old. Board exam is approaching (or already announced). High urgency. Arrived via Google search ("SSC গণিত MCQ"), Facebook group, or YouTube recommendation. Has a specific subject need.

**1. Immediate goal:** "I need SSC preparation material — MCQ practice, board questions, suggestions."

**2. Question in first 5 seconds:** "Does this have what I need for SSC? Is it organized by my syllabus?"

**3. Information needed immediately:** Confirmation that SSC content exists. Board question availability. MCQ count. Free vs. premium distinction for SSC content.

**4. Hero should encourage:** "SSC প্রস্তুতি শুরু করুন" (Start SSC preparation) or "বোর্ড প্রশ্ন দেখুন" (See board questions). The action should be SSC-specific.

**5. What should NEVER distract this visitor:** Class 6/7/8 content, HSC content, generic "best platform" claims, premium upsell before showing SSC content.

**6. Best CTA:** "SSC বোর্ড প্রশ্ন দেখুন" (See SSC board questions) — this is the highest-intent action for an SSC candidate.

**7. Next homepage section:** Exam Preparation (exam countdown + board question quick-start)

**Current drop-off risk:** VERY HIGH. The Hero is generic — it says "Class 6 থেকে HSC" which includes SSC but doesn't prioritize it. An SSC candidate arriving from a Google search for "SSC MCQ" lands on a generic page that doesn't confirm SSC content exists until they scroll to section 10 (ExamCountdown) or section 11 (BoardQuestion). That's 8-9 sections of irrelevant content before finding what they searched for.

### Visitor Type 3: HSC Candidate (First-Time)

**Profile:** 17-18 years old. University admission depends on HSC results. Highest academic stakes. Arrived via Google search ("HSC পদার্থবিজ্ঞান সমাধান") or coaching center recommendation. Sophisticated digital user.

**1. Immediate goal:** "I need HSC-level content — advanced MCQ, CQ solutions, model tests."

**2. Question in first 5 seconds:** "Is this advanced enough for HSC? Or is it just basic stuff?"

**3. Information needed immediately:** HSC content quality. CQ solution availability. Model test availability. Differentiation from basic platforms.

**4. Hero should encourage:** "HSC প্রস্তুতি শুরু করুন" (Start HSC preparation) or "মডেল টেস্ট দিন" (Take a model test).

**5. What should NEVER distract this visitor:** Class 6-8 content, basic MCQ without explanation, anything that feels "too simple" for HSC level.

**6. Best CTA:** "HSC মডেল টেস্ট" (HSC model test) — this signals advanced content.

**7. Next homepage section:** Exam Preparation (board questions for HSC level)

**Current drop-off risk:** HIGH. The Hero doesn't differentiate between class levels. An HSC candidate sees "Class 6 থেকে HSC" and wonders if the HSC content is afterthought. No HSC-specific proof in the Hero.

### Visitor Type 4: Parent (First-Time)

**Profile:** 35-50 years old. Found the platform via Facebook ad, friend recommendation, or Google search. Evaluating whether this is safe and useful for their child. Price-sensitive. Concerned about content quality and appropriateness.

**1. Immediate goal:** "Is this safe and useful for my child? Is it really free?"

**2. Question in first 5 seconds:** "What is this? Is it a legitimate education platform?"

**3. Information needed immediately:** Platform legitimacy (teacher credentials, institutional backing). Pricing transparency (what's free, what's paid). Safety indicators (no inappropriate content, secure payment).

**4. Hero should encourage:** "বিনামূল্যে শুরু করুন" (Start for free) — the lowest-risk action. Parents need to verify value before committing.

**5. What should NEVER distract this visitor:** Complex navigation, technical jargon, premium pricing without context, anything that feels "salesy" or unprofessional.

**6. Best CTA:** "ফ্রি কনটেন্ট দেখুন" (See free content) — parents want to verify before committing.

**7. Next homepage section:** Trust Building (teacher profiles, testimonials, "Why Sikkha")

**Current drop-off risk:** MEDIUM-HIGH. The Hero's particle animations and floating icons look "fun" but may signal "this is for kids, not serious education" to a parent. The word "premium" in the second CTA may create premature pricing anxiety.

### Visitor Type 5: Teacher (First-Time)

**Profile:** 25-45 years old. Education professional. Evaluating the platform for potential use with students or as a content contributor. Arrived via professional network, education conference, or colleague recommendation.

**1. Immediate goal:** "Can I use this as a teaching resource? Can I contribute content?"

**2. Question in first 5 seconds:** "Is this a serious educational platform or a student toy?"

**3. Information needed immediately:** Content quality and curriculum alignment. Teacher/contributor features. Institutional credibility.

**4. Hero should encourage:** "কনটেন্ট দেখুন" (See content) — teachers want to evaluate content quality before anything else.

**5. What should NEVER distract this visitor:** Student-oriented CTAs ("Start learning"), gamification elements, anything that feels "childish."

**6. Best CTA:** "কারিকুলাম দেখুন" (See curriculum) — teachers care about syllabus coverage.

**7. Next homepage section:** Subject Explorer (to verify curriculum coverage)

**Current drop-off risk:** MEDIUM. The platform looks student-focused, which may not immediately appeal to teachers. No teacher-specific pathway on the homepage.

### Visitor Type 6: Returning Student (Free User, Logged Out)

**Profile:** Has used the platform before. Knows the interface. Arrived with specific intent (needs to practice MCQ, check a board question, review a lecture). Likely on mobile.

**1. Immediate goal:** "I need to get to [specific content] quickly."

**2. Question in first 5 seconds:** "Where's the search bar? Where's my class?"

**3. Information needed immediately:** Search functionality. Quick navigation to their class/subject.

**4. Hero should encourage:** "Search" or "Go to your class" — the fastest path to content.

**5. What should NEVER distract this visitor:** Hero value proposition (they already know what the platform is), statistics, trust signals, premium upsell.

**6. Best CTA:** Search bar or "আমার ক্লাস" (My class)

**7. Next homepage section:** Quick Search or Subject Explorer

**Current drop-off risk:** LOW-MEDIUM. The returning student knows the platform but has to scroll past the Hero and AchievementBadges to reach search. The search bar is not immediately visible.

### Visitor Type 7: Returning Student (Free User, Logged In)

**Profile:** Same as above but authenticated. Has bookmarks, progress, and notes. Arrived via deep link or bookmarked URL.

**1. Immediate goal:** "Continue where I left off" or "Practice more MCQ."

**2. Question in first 5 seconds:** "Where's my dashboard? Where's my progress?"

**3. Information needed immediately:** Dashboard access. Recently viewed content. Progress summary.

**4. Hero should encourage:** "Go to dashboard" or "Continue learning" — personalized path.

**5. What should NEVER distract this visitor:** Generic Hero content (they've seen it), registration prompts, premium upsell (if they're not ready).

**6. Best CTA:** "ড্যাশবোর্ড" (Dashboard) or "শেষ দেখা কনটেন্ট" (Last viewed content)

**7. Next homepage section:** User Dashboard (bypass homepage entirely for logged-in users)

**Current drop-off risk:** LOW. The bottom nav provides dashboard access. However, the homepage doesn't personalize for logged-in users — they see the same generic Hero as first-time visitors.

### Visitor Type 8: Premium User

**Profile:** Has purchased premium content. Likely arrives via deep link to specific content. If landing on homepage, needs to access purchased content quickly.

**1. Immediate goal:** "Access my premium content" or "Check new premium additions."

**2. Question in first 5 seconds:** "Where's my purchased content?"

**3. Information needed immediately:** Dashboard. Purchased content list. New premium additions.

**4. Hero should encourage:** "Go to dashboard" or "View purchased content"

**5. What should NEVER distract this visitor:** Premium upsell (they already paid), generic content, registration prompts.

**6. Best CTA:** "আমার কনটেন্ট" (My content)

**7. Next homepage section:** User Dashboard

**Current drop-off risk:** LOW. Premium users are already converted. The risk is churn, not conversion.

### Visitor Type 9: User from Facebook

**Profile:** Arrived via Facebook post, group share, or ad. Likely mobile (80%+ Facebook traffic is mobile). Attention span is very short (Facebook scrolling mindset). May have clicked impulsively.

**1. Immediate goal:** "What is this? My friend shared it / I saw an ad."

**2. Question in first 5 seconds:** "Is this what the post/ad promised?"

**3. Information needed immediately:** Confirmation that the content matches the Facebook post/ad. Quick verification of legitimacy.

**4. Hero should encourage:** Match the Facebook post's promise exactly. If the post said "Free SSC MCQ practice," the Hero should say "SSC MCQ practice — free."

**5. What should NEVER distract this visitor:** Anything unrelated to the Facebook post's content, long text blocks, complex navigation, anything that requires thought.

**6. Best CTA:** Match the Facebook post's CTA. If the post said "Try free MCQ," the CTA should be "ফ্রি MCQ চেষ্টা করুন."

**7. Next homepage section:** Whatever the Facebook post promised (MCQ, board questions, lectures)

**Current drop-off risk:** VERY HIGH. Facebook traffic has the highest bounce rate. The generic Hero doesn't match any specific Facebook post's promise. The visitor has to figure out "is this what I clicked on?" which takes time they don't have.

### Visitor Type 10: User from Google Search

**Profile:** Arrived via specific search query (e.g., "SSC গণিত MCQ ২০২৪", "HSC পদার্থবিজ্ঞান সমাধান"). High intent. Has a specific need. Arrived on the homepage instead of a content page (common for new platforms without deep SEO).

**1. Immediate goal:** "Find what I searched for."

**2. Question in first 5 seconds:** "Is this what I searched for? Or did I land on the wrong page?"

**3. Information needed immediately:** Confirmation that the searched content exists. Search bar to refine.

**4. Hero should encourage:** "Search for [their query]" or "Find [their topic]"

**5. What should NEVER distract this visitor:** Generic content, unrelated sections, anything that doesn't match their search intent.

**6. Best CTA:** Search bar with their query pre-filled (ideal) or "বোর্ড প্রশ্ন খুঁজুন" (Search board questions)

**7. Next homepage section:** Quick Search (immediately accessible)

**Current drop-off risk:** VERY HIGH. Google search visitors expect to land on the exact page they searched for. Landing on a generic homepage is a mismatch. The search bar is 2 sections deep — by the time they find it, they may have already decided to go back to Google.

### Visitor Type 11: User from YouTube

**Profile:** Arrived via a YouTube video description link or channel link. Likely interested in video lectures. Arrived with expectation of video content.

**1. Immediate goal:** "Find the video lectures I saw on YouTube."

**2. Question in first 5 seconds:** "Where are the video lectures?"

**3. Information needed immediately:** Lecture availability. Video content confirmation.

**4. Hero should encourage:** "দেখুন লেকচার" (Watch lectures) or "ভিডিও কনটেন্ট" (Video content)

**5. What should NEVER distract this visitor:** Text-only content, MCQ practice (they came for videos), anything that doesn't match the video expectation.

**6. Best CTA:** "লেকচার দেখুন" (Watch lectures)

**7. Next homepage section:** Subject Explorer (to find the specific lecture topic)

**Current drop-off risk:** HIGH. The Hero doesn't mention video lectures. The word "lecture" appears in the subtitle but isn't prominent. A YouTube visitor looking for video content has to scroll and search.

---

### Visitor Type 12: Mobile Visitor (General)

**Profile:** 80%+ of Bangladeshi internet users are mobile-first. Slow internet connection (2G/3G common). Limited screen space. Thumb-driven navigation. Impatient with loading.

**1. Immediate goal:** Same as their visitor type above, but with mobile constraints.

**2. Question in first 5 seconds:** "Did this load? Can I use it on my phone?"

**3. Information needed immediately:** Content confirmation. Touch-friendly navigation. Fast loading.

**4. Hero should encourage:** Same as their visitor type, but with mobile-optimized CTA size and placement.

**5. What should NEVER distract this visitor:** Canvas particle animations (battery drain, performance hit), large images, anything that causes slow loading, horizontal scrolling.

**6. Best CTA:** Same as their visitor type, but with a full-width button (easier to tap).

**7. Next homepage section:** Same as their visitor type.

**Current drop-off risk:** HIGH for performance. The canvas particle system runs on every mobile visit, consuming battery and CPU. The 17-section homepage requires extensive scrolling on mobile.

---

### Visitor Type 13: Desktop Visitor (General)

**Profile:** Minority of traffic (likely 15-20%). Larger screen. Faster connection. More patient. May be a teacher, parent, or student using a school computer.

**1. Immediate goal:** Same as their visitor type above.

**2. Question in first 5 seconds:** Same as their visitor type.

**3. Information needed immediately:** Same as their visitor type.

**4. Hero should encourage:** Same as their visitor type.

**5. What should NEVER distract this visitor:** Mouse parallax (can be disorienting on large screens), mobile-optimized layouts that waste desktop space.

**6. Best CTA:** Same as their visitor type.

**7. Next homepage section:** Same as their visitor type.

**Current drop-off risk:** LOW. Desktop users are more patient and have more screen space. The 17 sections are less problematic on desktop.

## Part 3: Biggest Drop-off Points in Current Homepage

### Drop-off Point 1: Hero → Section 2 (AchievementBadges)
**Cause:** The visitor has just read the Hero. They're deciding whether to scroll. The next thing they see is statistics — the same statistics that were already visible in the Hero (4 stats in Hero + 6 stats in AchievementBadges). This is redundant and signals "the platform has nothing else to show."

**Impact:** Estimated 15-25% of visitors who scroll past the Hero will stop at Section 2. The redundant stats create a "is this it?" moment.

**Fix:** Remove AchievementBadges entirely. The Hero stats are sufficient.

---

### Drop-off Point 2: Section 2 (AchievementBadges) → Section 3 (QuickSearch)
**Cause:** The visitor has seen two sections of the same type (stats). They're now uncertain about what comes next. The search bar appears, but the visitor may have already decided to leave.

**Impact:** Estimated 10-15% additional drop-off. The visitor has scrolled through two sections without finding a clear next step.

**Fix:** Make search immediately visible after the Hero. No intervening sections.

---

### Drop-off Point 3: Hero CTA → Landing Page Mismatch
**Cause:** The Hero has two CTAs: "শিক্ষা শুরু করুন" (goes to class-list) and "প্রিমিয়াম দেখুন" (goes to premium). A visitor who clicks "Start learning" lands on a class list page. A visitor who clicks "See premium" lands on a premium page. Neither confirms the specific content the visitor was looking for.

**Impact:** Estimated 20-30% bounce rate after CTA click. The visitor expected specific content but landed on a generic listing page.

**Fix:** CTA destination should match visitor intent. SSC student clicking "Start learning" should land on SSC content, not a generic class list.

---

### Drop-off Point 4: Mobile Scrolling Fatigue (Section 6+)
**Cause:** On mobile, each section takes approximately one full screen height. By Section 6 (NoticeBoard), the visitor has scrolled through 5 full screens. The content becomes increasingly irrelevant to their initial intent.

**Impact:** Estimated 30-40% of mobile visitors stop scrolling after Section 5. Sections 6-17 are rarely seen on mobile.

**Fix:** Reduce to 6-8 sections. Remove sections that don't serve conversion.

---

### Drop-off Point 5: Premium Section Before Trust
**Cause:** PremiumBanner (Section 12) asks the visitor to consider paying before they've seen testimonials (Section 14) or teacher profiles (Section 13). The visitor hasn't built enough trust to consider payment.

**Impact:** Estimated 50-60% of visitors who see the premium section skip it. The premium CTA has low click-through because trust hasn't been established.

**Fix:** Move premium after trust signals.

## Part 4: Biggest Confusion Points

### Confusion Point 1: "Is this free or paid?"
**Evidence:** The Hero has two CTAs — one for "Start learning" and one for "See premium." The AchievementBadges section mentions "৮০% ফ্রি কনটেন্ট" (80% free content). The PremiumBanner is 12 sections deep. The visitor cannot quickly determine what's free and what's paid.

**Impact:** Visitors who can't determine pricing within 5 seconds assume the platform is paid and leave.

**Fix:** State "৮০% ফ্রি" prominently in the Hero. Separate free and paid pathways clearly.

---

### Confusion Point 2: "What class is this for?"
**Evidence:** The Hero says "Class 6 থেকে HSC" (Class 6 to HSC). This includes 6 different class levels. The visitor doesn't know if the platform prioritizes their specific class.

**Impact:** Visitors who don't see their class prioritized assume the platform isn't specialized for them.

**Fix:** If possible, detect class from URL/referrer and personalize. Otherwise, make class selection immediately visible.

---

### Confusion Point 3: "Where do I start?"
**Evidence:** The homepage has 17 sections, each with its own CTA. The Hero says "Start learning." Section 4 (QuickSearch) says "Search." Section 3 (SubjectExplorer) says "Select your class." Section 10 (ExamCountdown) says "Start preparation." Section 11 (BoardQuestion) says "Quick start." Section 17 (CTA) says "Register."

**Impact:** The visitor encounters 6 different "start" commands, each leading to a different destination. Decision paralysis.

**Fix:** One primary CTA. All other CTAs should be secondary (text links, not buttons).

---

### Confusion Point 4: "Why are there 3 statistics sections?"
**Evidence:** Stats appear in Hero (4 stats), AchievementBadges (6 stats), and EnhancedStats (5 stats). The same data is presented three times in different formats.

**Impact:** Repetition signals desperation, not credibility. The visitor wonders "why do they need to show me this three times?"

**Fix:** Statistics appear once, in context, with meaning.

---

### Confusion Point 5: "What's the difference between Board Questions and Exam Center?"
**Evidence:** BoardQuestionSection (Section 11) and ExamCountdownSection (Section 10) both relate to exam preparation. The visitor doesn't understand the distinction. "Board Questions" vs "Exam Center" — are these different things?

**Impact:** Visitors don't know which section to use for their exam preparation needs.

**Fix:** Merge into a single "Exam Preparation" section with clear sub-paths.

## Part 5: Elements That Interrupt Decision-Making

### Interrupting Element 1: Particle Canvas Animation
**What it does:** Runs a continuous animation loop on the Hero background, consuming CPU and battery.
**How it interrupts:** The moving particles distract from the text content. The visitor's eye follows the animation instead of reading the headline. On mobile, it causes jank and battery drain.
**Decision impact:** Delays comprehension by 1-2 seconds. On slow mobile devices, may cause the page to feel unresponsive.

---

### Interrupting Element 2: Floating Parallax Icons
**What it does:** 8 icons float around the Hero, moving with mouse position (desktop) or static (mobile).
**How it interrupts:** The icons create visual noise around the headline. The visitor's peripheral vision picks up the movement, pulling attention away from the text.
**Decision impact:** Dilutes the headline's impact. The visitor processes "what are those icons?" instead of "what does this platform do?"

---

### Interrupting Element 3: AchievementBadges Section
**What it does:** Shows 6 statistics in a section immediately after the Hero.
**How it interrupts:** The visitor has just processed the Hero. They're deciding whether to continue scrolling. The statistics section asks them to process 6 new data points. This is cognitive overhead at the exact moment they should be taking action.
**Decision impact:** Creates a "scroll or leave?" decision point that could be avoided by removing the section.

---

### Interrupting Element 4: SpecialNoticePopup
**What it does:** Opens a modal popup 1.2 seconds after page load.
**How it interrupts:** The visitor is processing the Hero. A popup appears, blocking the content. The visitor must either read the popup (distraction) or dismiss it (friction).
**Decision impact:** The popup is the single highest-friction element on the homepage. It forces an unwanted interaction before the visitor has even understood the platform.

---

### Interrupting Element 5: Multiple Competing CTAs
**What it does:** 5 different sections each have their own CTAs pointing to different destinations.
**How it interrupts:** Each CTA creates a micro-decision: "Should I click this? Or keep scrolling?" By the 5th CTA (section 11), the visitor has made 5 "click or scroll" decisions. This is exhausting.
**Decision impact:** Decision fatigue. The visitor stops making decisions and leaves.

---

### Interrupting Element 6: PremiumBanner Before Value Demonstration
**What it does:** Asks the visitor to consider premium content (Section 12) before they've seen testimonials (Section 14) or teacher profiles (Section 13).
**How it interrupts:** The visitor is in "explore" mode. The premium banner shifts them to "buy" mode prematurely. This mode-switch is jarring.
**Decision impact:** The visitor resists the premium pitch and may disengage from subsequent sections.

## Part 7: The Universal Hero Problem

Every visitor type analyzed above shares one common frustration with the current Hero:

**The Hero doesn't answer their specific question.**

| Visitor Type | Their Specific Question | Hero's Answer |
|-------------|------------------------|---------------|
| SSC Student | "Do you have SSC content?" | "Best education platform" (no) |
| Class 6 Student | "Is this free?" | "See premium" (no) |
| HSC Candidate | "Is this advanced enough?" | "Class 6 to HSC" (unclear) |
| Parent | "Is this legitimate?" | Particle animations (no) |
| Teacher | "Is this for professionals?" | "Start learning" (no) |
| Returning Student | "Where's the search?" | Hero value prop (no) |
| Facebook Visitor | "Is this what I clicked on?" | Generic content (no) |
| Google Visitor | "Where's what I searched for?" | Generic content (no) |
| YouTube Visitor | "Where are the videos?" | No video mention (no) |

**The core problem:** The Hero is designed for a hypothetical "average visitor" who doesn't exist. Every real visitor has a specific need, and the Hero doesn't address any of them.

**The universal fix:** The Hero must either:
1. **Personalize** (detect visitor type and tailor the message), or
2. **Generalize effectively** (state the platform's core value so clearly that every visitor type recognizes themselves)

Option 1 is ideal but requires technical implementation (class detection, referrer parsing, etc.). Option 2 is immediately achievable.

**The generalization formula:**
- Who: Class 6–HSC students
- What: Free exam preparation (MCQ, board questions, lectures)
- Why: 80% free, organized by syllabus, used by 10,000+ students

This covers every visitor type's core need: "Is this for me, does it have what I need, and can I trust it?"
